A Denormalized Database is a database that does not follow the normalization rules of a database
to the full extent to improve quicker data retrieval and query performance; in a normalized
database, the data integrity is saveed, and redundancy is avoided by organizing the columns and
tables based on their dependencies and relations. Normalized databases are ideal for transaction systems,
whereas a denormalized data sturcture is useful in reporting and analytical puposes.
The main advantages are - imporved query performance, Simplified Queries and optimization
